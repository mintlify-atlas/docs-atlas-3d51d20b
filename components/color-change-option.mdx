## Purpose

`ColorChangeOption` provides UI controls for customizing card visual properties:

- **Background type**: Color vs. Blur (song thumbnail)
- **Color picker**: Select solid background color
- **Blur index**: Choose which song thumbnail to blur
- **Hero display**: Character/emoji vs. Title text
- **Character input**: Custom character for hero display

This component is used within `CardCrafter` to configure playlist card appearance.

## Background Type Selection

### Color Mode

Displays a color picker when `bgType === "color"`:

```tsx
{crafterContext.inputCard.properties.bgType === "color" ? (
  <motion.div
    key="color"
    initial={{ opacity: 0, x: -10 }}
    animate={{ opacity: 1, x: 0 }}
    exit={{ opacity: 0, x: -10 }}
    transition={{ duration: 0.3 }}
  >
    <input
      value={crafterContext.inputCard.properties.color}
      onChange={(e) => {
        setCrafterContext((prev: any) => {
          return {
            ...prev,
            inputCard: {
              ...prev.inputCard,
              properties: {
                ...prev.inputCard.properties,
                color: e.target.value,
              },
            },
          };
        });
      }}
      type="color"
      style={{ backgroundColor: crafterContext.inputCard.properties.color }}
      className="aspect-square w-[1.8em] rounded-[0.7em]"
    />
    <p>Color</p>
  </motion.div>
) : (
  <Palette onClick={() => /* switch to color mode */} />
)}
```

### Blur Index Controls

```tsx
<button

        ...prev,
```

## Animation

All mode switches use Framer Motion for smooth transitions:

```tsx
<AnimatePresence mode="wait">
  {condition ? (
    <motion.div
      key="unique-key"
      initial={{ opacity: 0, x: -10 }}
      animate={{ opacity: 1, x: 0 }}
      exit={{ opacity: 0, x: -10 }}
      transition={{ duration: 0.3 }}
    >
      {/* Content */}
    </motion.div>
  ) : (
    <motion.div key="other-key" /* ... */>
      {/* Alternative content */}
    </motion.div>
  )}
</AnimatePresence>
```

The `mode="wait"` ensures one element exits before the next enters, preventing layout issues.

## Property Updates

All updates follow the same pattern:

1. Call `setCrafterContext` with updater function
2. Spread previous state
3. Update nested `inputCard.properties` object
4. Preserve all other properties

```tsx
setCrafterContext((prev: any) => {
  return {
    ...prev,
    inputCard: {
      ...prev.inputCard,
      properties: {
        ...prev.inputCard.properties,
        [propertyName]: newValue,
      },
    },
  };
});
```

This immutable update pattern ensures React detects changes and re-renders appropriately.

<Note>
  ColorChangeOption is tightly coupled with CardCrafter and expects the `CrafterContextType` structure. It cannot be used independently without this context.
</Note>

```tsx
function CardCrafter() {
  const [crafterContext, setCrafterContext] = useState({
    link: "",
    title: "",
    inputCard: {
      properties: {
        title: "My Playlist",
        color: "#3E3F76",
        blur: "1",
        bgType: "color",
        cdHero: "char",
        char: "🎵",
      },
      songs: [],
    },
  });
  
  return (
    <ColorChangeOption
      setCrafterContext={setCrafterContext}
      crafterContext={crafterContext}
    />
  );
}
```
