## Purpose

`Slider` is a styled HTML range input used for:

- Displaying current playback position
- Seeking to specific timestamps
- Visual feedback with custom thumb styling

Integrated with the `Player` component to control YouTube playback position.

## Props

<ParamField path="minimum" type="number" default="0">
  Minimum slider value (typically 0 for start of track)
</ParamField>

<ParamField path="maximum" type="number" default="100">
  Maximum slider value (track duration in seconds)
</ParamField>

<ParamField path="value" type="number" required>
  Current slider position (current playback time in seconds)
</ParamField>

<ParamField path="onChangeFn" type="(e: React.ChangeEvent<HTMLInputElement>) => void" required>
  Callback fired when user drags slider
</ParamField>

```tsx

  return (

      }}
    >
      <StyledWrapper className="w-full">
        <input
          style={{ width: "100%" }}
          type="range"
          min={minimum}
          max={maximum}
          value={value}
          onChange={onChangeFn}
          className="PB-range-slider"
          id="myRange"
        />
      </StyledWrapper>
    </div>
  );
};
```

## Thumb Styling

The slider thumb has:

- **Size**: `0.4em` wide × `5em` tall (vertical orientation)
- **Shape**: Fully rounded (`border-radius: 1000px`)
- **Color**: Red gradient from `#8c0b0b` to `#cf0808`
- **Shadow**: Inset shadow for depth
- **Hover**: Smooth transition on hover
- **Active**: Scales to 1.2× when dragging

## Seek Functionality

When the user drags the slider, it calls the `seekFn` from `serviceProvider`:

```tsx
import { seekFn } from '@/services/serviceProvider';

<Slider
  value={timestamp.current}
  onChangeFn={(e) => {
    seekFn(Number(e.target.value));
  }}
/>
```

The `seekFn` communicates with the YouTube iframe:

```tsx
// In serviceProvider.ts
const seekFn = (seconds: number) => {
  ytElement.command("seekTo", [seconds]);
};
```

## Value/onChange Interface

The slider follows controlled component pattern:

```tsx
interface SliderProps {
  value: number;                           // Current position (controlled)
  onChangeFn: (e: ChangeEvent) => void;    // Update handler
  minimum?: number;                        // Min value
  maximum?: number;                        // Max value
}
```

The parent component manages state:

```tsx
const [timestamp, setTimestamp] = useState({ current: 0, total: 0 });

<Slider
  value={timestamp.current}              // Read from state
  maximum={timestamp.total}
  onChangeFn={(e) => {                   // Update state
    const newTime = Number(e.target.value);
    seekFn(newTime);
  }}
/>
```

## Accessibility

HTML range inputs provide built-in accessibility:

- Keyboard control: Arrow keys adjust value
- Screen readers: Announce current value
- Focus indicators: Browser default outline

Consider adding:

```tsx
<input
  type="range"
  aria-label="Playback position"
  aria-valuemin={minimum}
  aria-valuemax={maximum}
  aria-valuenow={value}
  aria-valuetext={`${formatTime(value)} of ${formatTime(maximum)}`}
/>
```

## Integration Example

```tsx
import Slider from '@/components/Slider/Slider';
import { seekFn } from '@/services/serviceProvider';
import { useState } from 'react';

function CustomPlayer() {
  const [timestamp, setTimestamp] = useState({ current: 0, total: 300 });
  
  return (
    <Slider
      minimum={0}
      maximum={timestamp.total}
      value={timestamp.current}
      onChangeFn={(e) => {
        const newTime = Number(e.target.value);
        seekFn(newTime);
        setTimestamp(prev => ({ ...prev, current: newTime }));
      }}
    />
  );
}
```
